Team Introduction

This project was executed by Ben Chen and Otto Thomsen in August 2023, with the aim of assisting users in preemptively identifying malicious websites. The project's goals included enabling users to classify URLs prior to accessing websites, thereby enhancing users' cybersecurity and countering red-team cyber operations. Explanation of Tech Stack

We utilized two datasets from the Canadian Institute of Cybersecurity: 11,556 Malicious URLs and 35,378 Benign URLs.

Our tech stack includes a variety of tools and technologies:

  • Python for scripting and programming.
  • Google Colab and Replit for collaborative coding environments.
  • Word2Vec for word embedding.
  • NLTK for text tokenization.
  • Pandas for data manipulation and analysis.
  • HuggingFace for model uploading and sharing.
  • HTML, CSS, and JavaScript for frontend development.
  • Flask for creating the backend web application.

Demo + Explanation

In our project, we harnessed various features to predict whether a given URL is malicious or benign. These features included:

  • URL characteristics like scheme, netloc, path, params, query, and fragment lengths.
  • Presence of keywords associated with malware, phishing, attacks, and hacking attempts.

Our process for creating the classification model involved the following steps:

  1. Merging two dataframes: We combined the datasets containing malicious and benign URLs.
  2. Data Cleaning: We performed cleaning to ensure consistency and reliability of data.
  3. Data Preprocessing: Features were transformed and normalized for effective model training.
  4. Splitting Data: We divided the dataset into features (X) and the target (y).
  5. Feature Selection: We used the aforementioned features to predict the target column "malicious".
  6. Train-Test Split: The dataset was split into training and testing sets using the "train_test_split" function.
  7. Model Training: We utilized various sklearn models for training.
  8. Evaluation: Models were evaluated using relevant metrics to assess their performance.

Our KNN model, with neighbors set to 3, achieved an accuracy rate of approximately 96.77%.

Our project's ultimate goal was to provide users with a reliable tool to distinguish between malicious and benign URLs, thereby enhancing cybersecurity.
